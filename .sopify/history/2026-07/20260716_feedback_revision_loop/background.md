# 变更提案：反馈裁定闭环

## 需求背景

EvidentLoop `v0.1.0a0` 已能在 `audit.html` 中记录人工反馈，但反馈只能留在浏览器或导出为 JSONL，runtime 不会消费它。用户无法把裁定写回当前报告，也无法在同一份报告中继续下一轮反馈。

本方案补齐 code-diff profile 的最小反馈闭环，不扩张到模型复审、自动修复或通用反馈平台。

评分：

- 方案质量：9/10
- 落地验证：9/10

评分理由：

- 结果：用户流程、报告身份、人工裁定语义、失败边界和交付范围均已落地；schema `0.4` revision 和中断恢复均通过 fixture 与故障注入验证。

## 目标与用户收益

```text
在报告中裁定
  → 复制给 AI 更新报告
  → 粘贴给 AI coding 工具
  → EvidentLoop 更新原报告
  → 刷新后继续反馈
```

完成后，用户可以：

- 始终操作同一报告目录，不必管理多份默认修订副本。
- 在报告中区分“模型原判断 / 我的裁定 / 当前剩余问题”。
- 修改上一轮裁定，并在报告内部保留可追溯的 revision run。
- 在人工裁定清空剩余问题后看到候选通过，同时明确知道代码没有被模型重新审查。
- 让错误来源、过期反馈和冲突动作明确失败，而不是静默污染报告。
- 仅在明确要求“另存新报告”时，通过 `--out` 保留原报告并生成副本。

JSONL 导出继续作为高级、自动化和故障恢复入口，不作为主要 CTA。

## 已确认决策

1. 默认更新原报告；只有用户明确要求另存时才生成副本。
2. 人工裁定后可以得到 `pass_candidate`，但必须醒目标注“基于人工裁定，未重新审查代码”，并保留模型原结论。
3. 更新成功后必须得到匹配的 `audit.json + audit.html`；正常失败自动恢复旧产物对，突然中断则保留可恢复材料并在下次运行给出明确结果。
4. 主按钮使用“复制给 AI 更新报告”；JSONL 导出降为次要入口。
5. 来源不匹配、报告已变化或反馈冲突时停止，让用户刷新后重新复制；不自动猜测或合并。
6. 当前尚无外部用户，runtime 从本次变更起只接受 schema `0.4`；既有 `0.3` 报告只作为历史证据阅读，不建立兼容或迁移承诺。

以上产品决策已经收口。当前 schema 统一为 `0.4`；真实用户形成依赖后，再对后续版本承担兼容责任。

## 变更边界

本方案包含：

1. 反馈解析、身份校验、重复去除和冲突拒绝。
2. 人工事件持久化、当前结论重算和最小 run lineage。
3. 原目录内安全更新 `audit.json + audit.html`，以及显式另存副本。
4. “复制给 AI 更新报告”、Skill 定位、隐私提示和失败恢复。

明确不做：

- 模型复审、Prompt Eval、多审查者或自动修改代码。
- JSONL 导入 UI、批量修订或复杂历史界面。
- 跨工作区搜索、通用 workflow/event framework 或预设扩展点。
- 自动合并过期反馈、最后写入胜出策略或跨报告迁移器。
- 为本地单写者场景引入平台专用锁、系统级 crash-atomic 指针或新的报告存储层。
- 为 schema 升级增加兼容别名、迁移框架或同时维护两套反馈模型。

## 工程门禁

### Schema

fixture 已证明人机结论分层需要 schema `0.4`。当前 runtime、Skill、fixture 和活动文档只使用 `0.4`；已发布的 `0.3` tag、wheel 与报告保持原样，不进入当前 runtime。

### 成对更新与中断恢复

候选 JSON/HTML 必须在隐藏 sibling 中完整生成并校验。正常错误自动回滚旧产物对；突然断电或强杀进程时保留 candidate/backup。下次运行能唯一确认旧报告时自动恢复并提示用户刷新，状态不唯一时停止并列出可恢复路径，不自动猜测。

## 蓝图差异

现有蓝图要求反馈闭环不原地覆盖来源 `audit.json`。本方案按已确认用户心智调整为：用户继续使用同一路径，runtime 先生成新的 revision，再只成对替换 `audit.json + audit.html`；目录中的其他文件保持不变，也不丢失模型原判断和反馈链。实现完成后同步修改蓝图，不默认保留独立旧报告目录。

## 影响与风险

- 影响模块：feedback、revision、validation、renderer/JavaScript、CLI/API、Skill、文档与测试，预计约 10–14 个实现文件。
- 身份风险：用来源 audit SHA-256 与 `graph_id + run_id + target_id + fingerprint` 联合校验，并在提交前复核来源未变化。
- 结论风险：保留来源模型结论；人工裁定只形成新的当前结论，不冒充模型重新审查。
- 覆盖风险：候选 JSON/HTML 完整生成并校验后才替换；失败不得把半成品当作成功报告。
- 隐私风险：复制内容默认不含本地绝对路径，不自动上传反馈。
